# M486 - Cancel Objects

## Description

The M486 command cancels specific objects on the build plate, rather than cancelling the entire print job.

This macro handles this instruction gracefully.


</br></br>
## Params

| Param      | Description                           |
| ---------- | ------------------------------------- |
| `C`        | Cancel the current object             |
| `P<index>` | Cancel an object with a given index   |
| `S<index>` | Set the index of the current object   |
| `T<index>` | Reset state, set number of objects    |
| `U<index>` | Un-cancel object with the given index |


</br></br>
## Code Flow

Step 1: Check that `exclude_object` is enabled. If not, print an error.

```jinja
{% if 'exclude_object' not in printer %}
  {action_raise_error("[exclude_object] is not enabled")}
{% endif %}
```


</br></br>
Step 2: Check if a reset message has been sent

```jinja
{% if 'T' in params %}
  EXCLUDE_OBJECT RESET=1

  {% for i in range(params.T | int) %}
    EXCLUDE_OBJECT_DEFINE NAME={i}
  {% endfor %}
{% endif %}
```

</br></br>
This loops through the objects in the `T` parameter, and defines a name for each object.


</br></br>
Step 3: Check if a 'cancel current' command has been issued.

```jinja
  {% if 'C' in params %}
    EXCLUDE_OBJECT CURRENT=1
  {% endif %}
```

</br></br>
Step 4: Check if a 'cancel item' command has been issued.

```jinja
{% if 'P' in params %}
  EXCLUDE_OBJECT NAME={params.P}
{% endif %}
```

</br></br>
Step 5: Check if a 'set index' command has been issued

```jinja
{% if 'S' in params %}
  {% if params.S == '-1' %}
    {% if printer.exclude_object.current_object %}
      EXCLUDE_OBJECT_END NAME={printer.exclude_object.current_object}
    {% endif %}
  {% else %}
    EXCLUDE_OBJECT_START NAME={params.S}
  {% endif %}
{% endif %}
```

</br></br>
Step 6: Check if an 'un-cancel' command has been sent

```jinja
{% if 'U' in params %}
  EXCLUDE_OBJECT RESET=1 NAME={params.U}
{% endif %}
```
